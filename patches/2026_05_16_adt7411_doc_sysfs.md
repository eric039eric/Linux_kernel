## 背景說明

`Documentation/hwmon/adt7411.rst` 原本只列了三個 sysfs 屬性：

```
in0_input      vdd voltage input
in[1-8]_input  analog 1-8 input
temp1_input    temperature input
```

但 `drivers/hwmon/adt7411.c` 的實際實作，透過 `hwmon_ops.is_visible()` 與 `HWMON_CHANNEL_INFO()` 巨集，已經完整支援電壓 min/max/alarm、溫度 min/max/min_alarm/max_alarm，以及外部溫度感測器的 fault 屬性。  

此外，原本 Notes 段落有一句：  

> SPI, external temperature sensor and limit registers are not supported yet.

這句描述和 driver 的實際實作已經不符合，因此一併修正。

---

## 分析過程

### 確認 driver 實作

```bash
sed -n '210,640p' drivers/hwmon/adt7411.c
grep -n 'HWMON_CHANNEL_INFO\|is_visible\|adt7411_read\|adt7411_write' drivers/hwmon/adt7411.c
```

主要發現：

- `HWMON_CHANNEL_INFO(in, ...)` 包含 `HWMON_I_INPUT | HWMON_I_MIN | HWMON_I_MAX | HWMON_I_ALARM`
- `HWMON_CHANNEL_INFO(temp, ...)` 包含 `HWMON_T_INPUT | HWMON_T_MIN | HWMON_T_MAX | HWMON_T_MIN_ALARM | HWMON_T_MAX_ALARM`，以及第二個 temp channel 額外有 `HWMON_T_FAULT`
- `adt7411_is_visible()` 控制 `temp2_*` 與 `in1`/`in2` 的可見性，條件是 `data->use_ext_temp`

### 確認 sysfs-Interface 標準命名

```bash
grep -n 'in\[0-\*\]_min\|temp\[1-\*\]_min_alarm' Documentation/hwmon/sysfs-interface.rst
```

確認命名符合 kernel 標準 hwmon sysfs 介面規範。

## 變更內容

### 修改檔案

`Documentation/hwmon/adt7411.rst`

### 主要改動

1. **補充主表格**：新增 `in0_min/max/alarm`、`in[1-8]_min/max/alarm`、`temp1_min/max/min_alarm/max_alarm`
2. **新增條件區塊**：外部溫度感測器啟用時才出現的 `temp2_input/min/max/min_alarm/max_alarm/fault`
3. **補充說明句**：外部溫度感測器啟用時，`in1` 和 `in2` 不可用
4. **修正 Notes**：移除 `external temperature sensor and limit registers are not supported yet`，只保留 `SPI is not supported yet.`

## Commit Message

```
Documentation: hwmon: adt7411: document supported sysfs attributes

The adt7411 driver exposes additional standard hwmon attributes beyond
the ones currently listed in Documentation/hwmon/adt7411.rst.

Document voltage min/max/alarm attributes, temperature min/max and
min_alarm/max_alarm attributes, and the temp2_fault attribute for the
external temperature channel.

Also update the documentation to clarify that analog inputs in1 and in2
are not available when the external temperature sensor is enabled, and
remove the outdated statement claiming that external temperature support
and limit registers are unsupported.

Signed-off-by: Chen-Shi-Hong <eric039eric@gmail.com>
```
